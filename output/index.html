CRYPTARSI

About

CRYPTARSI is a system designed to produce, search and browse encrypted data sets. The data can be hosted on unsecured servers or locally, inside your browser local storage. The name comes from the Latin word "Crypto" and the Bulgarian word "Tarsi", meaning "search".

Cryptarsi is using encrypted raw text files. There is no scripting, no RDBMS engine behind, the simpliest web server is enough to put online your searchable encrypted data.

Cryptarsi is available on Github. The old development code is found under the name "Mihalya". An early version of the system was used by the journalists from Bivol.bg while working on the Cablegate releases of Wikileaks from different locations.

The system may be useful in different fields, requiring data protection: investigative journalism, human rights activism in dictatorial countries, corporate security, etc.

If you want to contribute to the development of Cryptarsi join us on Github or donate a small amount.

Requirements: The search interface of Cryptarsi works well with the latest Firefox and Chrome and to some extent with Safari. The encoder requires Perl environment with some extensions installed. Read more how to use the encoder

Back to reality, the actual Cryptarsi encoder is available from Github https://github.com/bivol/cryptarsi and is working fine, but you need a Perl environment to run it. Here is how:

A. Prepare your data

Cryptarsi can process UTF-8 formated text files. If the file contains only text, they are considered a single document. You can put multiple documents in one text file. In this case you have to separate them with

=======================DATA ENDS============================

Take a look at the data.txt file in the source folder to figure out how to prepare and structure your data.

If you want to attach files to a particular document there are two mandatory steps. Your document must have a unique refid and all the files must be in a folder named after the refid. The source folder and data.txt give such an example for two documents with the refid: CT007 and refid: CT008 Cryptarsi accepts audio files (mp3 and ogg), picture files (jpg and png) and PDF files.

OK, your source folder is ready, now let's process it.

B. Processing your data

On Windows:
  • Download Strawberry Perl and install it http://strawberryperl.com/
  • Open the unzipped Cryptarsi folder and double click on the cryptarsi.pl file
  • At the terminal prompt provide a username, password, the source folder (with the files you prepared) and the output folder. The script will do the job and exit.
On Mac OS:
  • Open a terminal (Applications->Utilities->Terminal) and type: 
    sudo cpan install Crypt::OpenSSL::AES
    Enter your user password. The system will prompt you to install some additional command line tools. Install them and run this command again. This will install the crypto library required by Cryptarsi. Then type: 
    sudo cpan install File::Copy::Recursive

    If everything goes well you will finish with the right perl modules installed.

On Linux:
Use cpan to install the required libraries 
Crypt::OpenSSL::AES
and 
File::Copy::Recursive

C: Explore or distribute the data

  • On the Web: You can transfer the output folder to any web hosting and point your browser to the index.html file.
  • Run from a local web server. There are multiple solutions to run a simple web server on your computer. Take a look at the free edition of Mongoose web server for Windows and Mac or google for a WAMP, MAMP, LAMP distribution for your system.
  • From a disk or usb stick. You can use the latest Firefox with modified settings for accessing local files. In the address field type 
    about:config
    , agree with the warning then search for fileuri and set the 
    security.fileuri.strict.origin_policy
    to false. While you are there you can also increase the size of the localstorage. Search for 
    dom.storage.default.quota
    and change the value to 20 or more MB to have enough space for indexes. If you use the Portable edition of Firefox you can save the settings and distribute the Cryptarsi data together with the modified browser on a usb stick.

Search results will display here. Take note of some search hints.

The default query will return the documents matching the words in the same order. Example: Investigative Reporting will return the string « Investigative Reporting », also « Investigative journalists started reporting », but not « Reporting was done by investigative journalists »

You can use + and - switches to include or exclude words from the query without matching the words order. Thus +Investigative +Reporting will return both examples above, but +Investigative +Reporting -done will return only the first example.

If you want to search for a particular date enter the date in european format without ponctuation: 09 10 2007 (October 9-th 2007).

Very important! Don't use the browser back button as it will bring you back to the login page. Use the search button to return to search results.

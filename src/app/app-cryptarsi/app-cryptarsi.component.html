Create, query and share encrypted data CRYPTARSI - a browser based files encryption and search engine

Investigative journalists; human right activists and many other professionals need to protect data and to share protected data easily. Cryptarsi is a system designed to meet those needs without installing a specific software.

Cryptarsi is designed to work straight from the browser. If you read this on Chrome browser, then you can start using Cryptarsi right now. Otherwise install Chrome or Chromium and open this url again. Cryptarsi is loading from internet, but your data and passwords stay on your computer and never travel through internet. You can even disconnect your computer from internet now and continue using Cryptarsi.

With Cryptarsi you can easily encrypt a single file or a large collection of data: text, PDF, audio and video files and view them without leaving your browser. Text files are indexed and searchable. If you take some time to describe your data or if you have a big cache of text data, Cryptarsi is able to search and locate the relevant documents and files.

Cryptarsi is an open source project available on Github. The development of the last version of Cryptarsi was supported by Google Digital News Initiative Fund.

The name Cryptarsi comes from the Latin word "Crypto" and the Bulgarian word "Tarsi", meaning "search".

If you want to contribute to the development of Cryptarsi join us on Github.

An early version of the system was used by the journalists from Bivol.bg while working on the Cablegate releases of Wikileaks from different locations.

File types support in Chrome: Text UTF-8 (.txt .html .xml); Images: jpg, png, gif; Audio: wav, ogg, mp3; Video: mp4, webm. Other files are downloadable and can be opened in external viewers. Text files are indexed and searchable.

Requirements: Chrome or Chromium browsers. The Firefox version is not fully operational.

Limitations: Because of a bug in CryptoJS a single file should not be bigger then 100 megabytes.

For the impatients... Fast start

The most simple usage of Cryptarsi is to encrypt files and to consult them or share them with other users. From the top of the sidebar click on the button. Select a name for your data set, give it a description, type a strong password, click on "Add files", select individual files or a directory full of files from your disk and click again on the red button to import them.

Voil√†, you have a new encrypted data set and his name will appear in the left menu.

To see the content of the newly created data set click on the name, type the password and type on the list button next to the search button. to display the list of all the files. Click on a file name to open it in a new tab where you can view or download the file.

The encrypted data you just created is easy to export and share with other Cryptarsi users. To do this open the data set, unlock it with the password and click on the Export Data button

The Cryptarsi export will produce a .tar archive with the encrypted data. You can send the tar file through email or, if it is too big, share it in the cloud. Send the password to the persons allowed to use the archive by a separate and secured channel: encrypted chat or email.

An even better practice is to transmit the tar archive through the anonymous Tor network using Tor Browser and Onionshare. Then send the password through the Ricochet anonymous instant messaging. Install those state of the art tools and start using them now to protect your privacy and your sources.

To import a Cryptarsi data set from file click on Import Data. Type a name for the data set, select the tar file and click the Import button.

To give it a try without creating a new data set just download this file with a demo data set and import it. Select the newly imported data set from the list. The password for the demo data set is simply: test.

Go to the Search field and type "cryptarsi". As you can see you can do much more with Cryptarsi then simply encrypt files.

Prepare your data Searching encrypted data with Cryptarsy

Cryptarsi has a powerful search engine and you can work with hundreds and even tousends of text files. Cryptarsi will automatically index any text in UTF-8 format found in files with .txt extension. Each text file can have multiple attachments - pdf, images, audio and even video files. Follow this guide to prepare your data for meaningful searches.

Data structure

The Cryptarsi data is structured a bit like an email: a plain text and (a lot of) attached documents. The text will be indexed, so the content you put in is important. You will later search and find your documents by the keywords in the texts. We use plain text files (UTF-8 encoded) and a mandatory .txt extension. You can produce such files with any writing app.

First, create a folder for your story. Put there the text files for each lead in the story. It should look like this: 

My story
|
|--Lead1.txt
|--Lead2.txt
|--Lead3.txt
Then suppose you have three documents you want to attach to your Lead1.txt. Create a floder named Lead1 (exactly as the text file but without the .txt extension. Put your files inside. It should look like this: 
|--Lead1
|	|--Lead_1_supporting_doucuments.pdf
|	|--Picture_for_lead_1.jpg
|	|--The_Sound_Recording_for_Lead1.mp3
And the whole folder looks like this: 
My story
|
|--Lead1.txt
|--Lead1
|	|--Lead_1_supporting_doucuments.pdf
|	|--Picture_for_lead_1.jpg
|	|--The_Sound_Recording_for_Lead1.mp3
|--Lead2.txt
|--Lead3.txt

You can also attach a single document from a different type then text in the root of MyStory. But the system will recognize one file of each type if you name it exactly as the .txt file. For example, if you have one mp3 sound file and one png pic file for Lead2.txt you must name them accordingly Lead2.mp3 and Lead2.png Like this:

My story
|
|--Lead1.txt
|--Lead1
|	|--Lead_1_supporting_documents.pdf
|	|--Picture_for_lead_1.jpg
|	|--The_Sound_Recording_for_Lead1.mp3
|	|--Other_Document_for_Lead_1.doc
|	|--etc...
|--Lead2.txt
|--Lead2.mp3
|--Lead2.png
|--Lead3.txt

Attachments are not mandatory. you can have a story with multiple text files or just one text file.

Here you can download a file with different data sources you can use for testing purposes.

Importing data from directory

The story folder will be imported and indexed in Cryptarsi with the button Create new data set. Give a name to the data set, write a strong password, select the folder with the story and click on "Encrypt and Import". The data set will then appear in the Menu on the left side of the app.

Searching in the data set

First, select a data set from the list and type the password to open it for search.

If you just type a sequence of words, the results will match all the text documents having the words in the same order. If you type the sequence in quotes, the exact expression will be returned. If you type + before the words all the documents having those words regardless the order will be returned. If you type - (minus sign) before a word documents having this word will be excluded from the search.

To understand the syntax of the search let's say we have 3 documents. Each document has one of the following expressions:

A.We need investigative reporting
B.Investigative journalists started reporting
C.Reporting was done by investigative journalists

So, if you type Investigative Reporting the system will return the string We need investigative reporting and Investigative journalists started reporting, but Reporting was done by investigative journalists will not be found.

"Investigative reporting" with quotes will match only A. Investigative journalists started reporting and Reporting was done by investigative journalists will not be found.

+Investigative +Reporting will return both A,B and C, but +Investigative +Reporting -done will return only A and B.

If you want to search for a particular date enter the date in european format without ponctuation: 09 10 2007 (October 9-th 2007).

Share your data Exporting and importing data

Once you have build a data set from your data you can easily export it and share it with other Cryptarsi users.

To do this open the data set, unlock it with the password and click on Export Data.

The export will produce a .tar archive containing encrypted data and indexes. You can send the tar file through email or if it is too big, share it in the cloud. Send the password to the persons allowed to use the archive by a separate and secured channel: encrypted chat or email. An even best practice is to split the password and to send the parts through different secured channels.

To import a Cryptarsi data set from file click on Import Data. Type a name for the data set, select the tar file and click Import. If the import is successful the data set will appear in the list.

It is a good idea to export all your data sets as tar files for archiving purposes. You can copy a CrypTARsi archive on a memory stick and use it on any computer having a Chrome browser installed.

If the imported data set has hundreds of documents the import may be slow. Take a cake and wait a minute or two! The documents need time to be indexed properly. The import is successful when the data set name appears in the list and you can log in.

You can also import some test data from this file. The password for the test data set is simply: test.

Protect your data Security matters

Cryptarsi is vulnerable if somebody can sniff or guess the passwords you use to encrypt the data. Your password and data will never travel through the net while you are working with Cryptarsi. Keep in mind, that after the Cryptarsi page is loaded in the browser you don't need the internet connection any more. You can even disconnect from internet, but don't reload the page and don't hit the back button while you are disconnected, because the app will break.

Do not leave open the data set you are working on while you are away from your computer. Click on the button to lock the data set anytime you want to avoid eyesdropping.

To delete all traces of your work click on Close data set, then wait a bit and click on Drop data set. This will erase the data and index files from the app memory.

It is safe, while not recommended, to send the Cryptarsi tar files through unsecured connections, because they are already encrypted. But never send a password for a cryptarsi tar file by unprotected email or any not secured chat.

The best practice is to transmit the tar archive through the anonymous Tor network using Tor Browser and Onionshare. Then send the password securely. PGP encrypted email will be fine. You can also use the Ricochet anonymous instant messaging. On mobile devices use the app Signal by Open Whisper system. Install those state of the art tools and start using them now to protect your privacy and your sources.
